# Granat UI — working rules

## Структура и подход
- Стек: React 18 + Vite, структура каталогов как у PrimeReact.
- Компоненты лежат в корне проекта по папкам (`button/`, `spinner/` и т.д.).
- Каждый компонент экспортируется в корневом `index.ts`.

## Стили и токены
- Только наши подключенные стили из `resources/`.
- Без inline‑стилей. Все правила — в `.css` файлах компонента или в `dev/storybook/`.
- Если нет нужного токена — добавляем локальную CSS‑переменную в компонент.
- Маппинг Figma → CSS токены напрямую по именам:
  - `Brand/MTS Red` → `--color-brand`
  - `Icons/Primary` → `--color-icon-primary`
  - `Icons/Secondary` → `--color-icon-secondary`
  - `Constant/Greyscale 0` → `--color-constant-greyscale-0`
  - `Constant/Greyscale 900` → `--color-constant-greyscale-900`

## Иконки
- Используем только иконки из `icons/`.
- Если в Figma есть имя/папка, стараемся найти одноимённую иконку у нас.

## Пропсы
- Компоненты должны быть максимально параметризованы пропсами.
- Дефолты соответствуют дизайн‑системе.

## Dev / Storybook
- `npm run dev` запускает визуальный сторибук.
- Сторибук:
  - `dev/storybook/` — оболочка, меню, стили
  - `dev/stories/` — истории компонентов
- После разработки каждого компонента добавляем его историю.
- Сайдбар содержит список по папкам компонентов.

## Темы
- Сейчас активна тема light через `data-mts-theme="light"` на `html`.
